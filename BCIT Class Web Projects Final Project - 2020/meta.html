List of Changes

Site Map

To view the planned architecture of the new site, please click here.

The structure has been created to reflect the changes and updates made for this project. The content of each page has been changed to group together the topics of each lecture, which will make it easier for search engines to view. The navigation bar will serve as a hub for all of the necessary links on the page, to all the lectures, resources and information that exist on the site.

Global

On a global scale, changes were made to increase user accessibility, decrease browser speed and establish forward compatiblity for updates in order to increase user traffic and improve user experience. Students are the primary users of these webpages so it is important that any student is able to access the site on a variety of devices and that they can access the information quickly and efficiently to help them with their studies.

  • Pages will now use the latest version of Hypertext Markup Language(HTML) - HTML5:
    • Compatible with most browsers, easier for users to access site.
    • Simpler than previous versions, decreases amount of code required and improves browser speed.
    • New elements provided by HTML5 are easier for search engines to recognize, pages are more likely to appear in search results.
  • Pages will also include one global pre-processor Syntatically Awesome Style Sheet (SASS) to write CSS3 style sheet:
    • Allows values (one or multiple) to be stored as variables, decreases repeat code and improves browser speed.
    • Contains a variety of features not included in regular style sheets, ensures forward compatiblity for future changes.
  • Pages now include media queries for mobile and tablet devices:
    • Increases user traffic as content is well-styled and formatted for each type of device.
  • Media query for printers is now included:
    • Instead of including every part of the website, only the important information (the title of the page and the course notes) will be available for users to print.
    • Decreases the amount of pages of content that will print, saving the user resources.
  • Re-styled the layout and changed the colour scheme:
    • Pages are more accessible, easier to accomodate the variety of users on the site.
    • Colour scheme helps make the site stand out among BCIT's web pages, refreshing for students studying from the notes online.

Header

Changes to the header involved storing the basic structure and content into a separate file and removing or re-locating links. This helps to increase user accessibility, decrease browser loading time and direct the user to focus more on the main content.

  • Structure and content for header stored in a PHP file:
    • Decreases the amount of repeated code as every page requires the header, contributes to increased browser speed.
  • Removed links to other BCIT pages and other class sections:
    • Links to other class sections are redundant, content for each class is similar and if a wrong link is clicked, users can instead click on the BCIT logo in the header to view all course sections.
    • Links to other BCIT pages moved to navigation bar, easier to access than previous location to increase user accessibility.
  • Removed page title and grouped with main content, easier for search engines to locate associated content.

Navigation

Navigation bar is now in a fixed position on screen and contains links for each page on the site, including links to other BCIT pages and resources. Redundant links and features have also been removed to help improve user experience and accessibility.

  • Structure and content for the navigation bar stored in a PHP file:
    • Decreases the amount of repeated code as every page requires the header.
  • Navigation bar re-located beside main content, easier to access every page on site.
  • Links to lectures no longer in drop-down menu, every link displayed inside navigation bar:
    • Clicking original drop-down menu an unnecessary step, displaying all lectures increases user accessibility.
    • Displaying all links to lecture replaces the 'Top of Page' link in footer, users can access pages more quickly from any part of the main content.
  • Link to home page of course moved inside navigation bar:
    • Easier to access than previous location at the top-left of screen, increasing user accessibility.
  • Navigation bar is now in a fixed position on screen:
    • Users can access any part of the site from any location in the main content, improves user experience.

Main Content

The main layout of the page no longer has the main content use the entire space, decreasing the amount of white space and allowing the navigation bar to be more accessible. HTML5 semantics are an important addition to increase search engine optimization and therefore be more likely to appear in search results which can increase user traffic.

  • Content divided into sections tags with appropriate headings using HTML5:
    • Increases search engine optimization.
    • Better organization of information, future updates are easier to manage in the backend.
    • Headings for each section are more visibly distinct, easier for users to follow content.
  • Page space is allocated for the navigation bar, main content no longer produces as much white space, increasing user experience.

Footer

Featured links in the footer were removed along with the footer as they are redundant or hardly used. Decreases the amount of code needed per page to help increase browser speed and deliver content to users quickly and efficiently.

  • Removed the footer - unnecessary links:
    • 'Top of Page' link is no longer required as the navigation bar displays every page, link is redundant.
    • 'Validate HTML' link is not useful, validates the current page's HTML which is not important for the curriculum.
